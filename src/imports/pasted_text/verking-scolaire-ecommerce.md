قم ببناء منصة تجارة إلكترونية احترافية جدًا، كاملة، وقابلة للتشغيل الفعلي، خاصة بعلامة تجارية متخصصة في الأدوات المدرسية.

==================================================
نوع المشروع

هذا ليس موقعًا بسيطًا.
هذا ليس نموذجًا تجريبيًا.
هذا يجب أن يكون نظام تجارة إلكترونية مركزيًا، احترافيًا، وقابلًا للتوسع، يتكون من:

1. موقع ويب للزبائن
2. تطبيق موبايل / APK للزبائن
3. لوحة تحكم إدارة مركزية
4. Backend / API حقيقي وآمن
5. قاعدة بيانات + تخزين صور + نظام تسجيل دخول
6. مزامنة فورية بين لوحة الإدارة والموقع والتطبيق

المطلوب الأساسي جدًا:
أريد لوحة إدارة واحدة مركزية أتحكم منها في:

- المنتجات
- الصور
- الأسعار
- المخزون
- الألوان
- البنرات
- الأقسام
- المحتوى
  ويظهر كل ذلك مباشرة في:
- الموقع
- التطبيق

الهدف الرئيسي

أريد منصة تجارة إلكترونية عالمية المستوى بحيث:

- الزبون يتصفح المنتجات من الموقع أو التطبيق
- المدير يتحكم في كل شيء من لوحة إدارة واحدة
- كل المحتوى يكون Dynamic وليس ثابتًا
- كل صور المنتجات يرفعها المدير من لوحة الإدارة
- الأسعار والمخزون يتغيران من لوحة الإدارة
- المدير يستطيع التحكم في ألوان الموقع والخلفيات والبنرات والهوية
- نفس الـ Backend ونفس قاعدة البيانات يخدمان الموقع والتطبيق معًا

لوحة الإدارة

قم ببناء لوحة إدارة فائقة الاحتراف، تكون مركز تحكم حقيقي للشركة.

يجب أن تحتوي على:

1. لوحة المؤشرات الرئيسية

- إجمالي الأرباح
- عدد الطلبات
- عدد العملاء
- تنبيهات نقص المخزون
- أكثر المنتجات مبيعًا
- آخر الطلبات
- طلبات الجملة المعلقة

2. إدارة المنتجات

- إضافة / تعديل / حذف
- رفع عدة صور
- إدارة الألوان والمقاسات إن وجدت
- تحديد المخزون
- تحديد السعر والتخفيض
- تحديد منتج جديد / مميز / الأكثر مبيعًا
- تفعيل / تعطيل المنتج

3. إدارة التصنيفات

- إنشاء / تعديل / حذف
- رفع صورة أو بانر للتصنيف
- ترتيب التصنيفات

4. إدارة الطلبات

- عرض جميع الطلبات
- تفاصيل الطلب
- تغيير حالة الطلب:
  - جديد
  - مؤكد
  - قيد المعالجة
  - تم الشحن
  - تم التسليم
  - ملغى
- رؤية طريقة الدفع
- رؤية بيانات الزبون
- تصفية الطلبات وتصديرها

5. إدارة العملاء

- قائمة العملاء
- نوع العميل: تجزئة / جملة
- تاريخ الطلبات

6. إدارة طلبات الجملة

- استقبال الطلبات
- مراجعتها
- تغيير حالتها
- متابعة التواصل مع الزبون

7. تخصيص الهوية والثيم (مهم جدًا)

- تغيير اللون الأساسي

- تغيير صور البنرات الرئيسية

- تغيير لوجو العلامة
- تغيير النصوص التسويقية
- إظهار/إخفاء أقسام الصفحة الرئيسية
- إدارة Theme Presets

وهذا التخصيص يجب أن ينعكس على:

- الموقع
- التطبيق حيثما أمكن

8. إدارة المحتوى

- من نحن

- العنوان

تدفقات العمل الحقيقية

FLOW 1
المدير يضيف منتجًا مع الصور والسعر والمخزون والتصنيف
→ يظهر المنتج تلقائيًا في الموقع والتطبيق

FLOW 2
الزبون يتصفح من الموقع أو التطبيق
→ يضيف للسلة
→ يطلب
→ يُحفظ الطلب في قاعدة البيانات
→ يراه المدير فورًا في لوحة الإدارة

FLOW 3
المدير يغيّر السعر أو الصور أو المخزون أو البنرات أو الألوان
→ تنعكس التغييرات تلقائيًا في الموقع والتطبيق

FLOW 4
الزبون يرسل طلب جملة
→ يصل إلى المدير في لوحة التحكم
→ يتم التعامل معه هناك

FLOW 5
الزبون يختار وسيلة الدفع
→ يتم حفظ الطلب مع حالة الدفع
